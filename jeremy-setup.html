1

Create a GitHub account & get access Sarah does this

The path-finder code is stored on GitHub (a website for sharing code). You need a free account to download it.

  1. Go to github.com and click Sign up — use any email, pick any username.
  2. Confirm your email when GitHub sends you a verification message.
  3. Tell Sarah your GitHub username. She'll add you to the Field Trips organisation.
  4. You'll get an invitation email from GitHub — click Accept invitation.
Wait before continuing Don't go to Step 2 until you've accepted Sarah's invitation. The download in Step 2 will fail without it.
Already have a GitHub account? Just send Sarah your username. Skip making a new one.
2

Download the code

This one command creates a field-trips folder on your Mac and downloads the path-finder into it:

mkdir -p ~/field-trips && cd ~/field-trips && git clone https://github.com/Field-Trips/path-finder.git && cd path-finder

You'll see lines like Cloning into 'path-finder'... then Resolving deltas: 100%. Takes about 5 seconds.

✅ Check it worked Paste this and press Enter:
ls ~/field-trips/path-finder/src/
You should see path_finder.py in the result.
❌ If you see "Repository not found" You haven't accepted Sarah's GitHub invitation yet. Check your email, accept it, then try again.
3

Set up the Python environment

This creates a private workspace for the path-finder's dependencies (extra libraries it needs) and installs them. Paste all of this at once:

cd ~/field-trips/path-finder && python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt

Takes 1–2 minutes. You'll see a lot of Downloading... and Installing... lines — that's normal. It ends with Successfully installed...

✅ Check it worked
cd ~/field-trips/path-finder && .venv/bin/python -m src.path_finder --help
You should see: Usage: python -m src.path_finder [OPTIONS] COMMAND [ARGS]...
❌ If you see errors about packages not found Run this to retry the install:
cd ~/field-trips/path-finder && source .venv/bin/activate && pip install -r requirements.txt
4

Add your secret keys Sarah sends these

The path-finder needs two keys to work — one to call the AI that picks each Wikipedia hop, one to save results to the shared database. Sarah will send both to you over iMessage.

First, create the secrets file:

cd ~/field-trips/path-finder && cp .env.example .env && open -e .env

TextEdit will open. The file looks like this:

ANTHROPIC_API_KEY= SUPABASE_URL=https://vjikcsifkvphuiwjrmqi.supabase.co SUPABASE_SERVICE_ROLE_KEY=

Fill in the two blank lines with the values Sarah sends you. It should look like:

ANTHROPIC_API_KEY=sk-ant-api03-... SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...

Save with ⌘S and close TextEdit.

⚠️ Important No spaces around the = sign. KEY=value is correct. KEY = value will not work.
✅ Check the database connection
cd ~/field-trips/path-finder && .venv/bin/python -c " from dotenv import load_dotenv; load_dotenv(override=True) import os from supabase import create_client from supabase.client import ClientOptions sb = create_client(os.environ['SUPABASE_URL'], os.environ['SUPABASE_SERVICE_ROLE_KEY'], options=ClientOptions(schema='path_finder')) rows = sb.table('nodes').select('title').limit(5).execute().data print('Connected. Nodes in database:', len(rows)) for r in rows: print(' -', r['title']) "
You should see: Connected. Nodes in database: 3 with article titles listed.
❌ If you see an authentication error Open .env again (open -e ~/field-trips/path-finder/.env) and check:
• No space between = and the key value
• The full key is there — they're long, make sure nothing got cut off
5

Run your first path

Everything is set up. Start the interactive runner:

cd ~/field-trips/path-finder && python run.py

It'll ask you three questions:

  1. Start Wikipedia article — for your first test, type Monhegan
  2. End Wikipedia article — type Conservation movement
  3. Permutations — type 1

Then press Enter and wait 30–90 seconds. You'll see the path it walked:

[1] ✓ via coastal island ecology Monhegan, Maine → Protected area → Conservation movement

That path is now in the shared database — both you and Sarah can see it.

✅ You're done From now on, whenever you want to run the path-finder, just open Terminal and paste:
cd ~/field-trips/path-finder && python run.py

Getting updates when Sarah changes something

Takes a few seconds. Do this anytime Sarah says there's an update.

?

If something goes wrong

"Wikipedia page not found"
The article title must match Wikipedia exactly. Open Wikipedia in your browser, find the article, and copy the title from the browser tab word-for-word.

"No such file or directory"
You need to navigate to the right folder first. Run: cd ~/field-trips/path-finder then try again.

The result shows ✗ instead of ✓
The path hit its hop limit before reaching the end article. Try a closer pair of topics, or run with a higher limit by adding --max-hops 15.

Anything else
Copy the error message and send it to Sarah.
