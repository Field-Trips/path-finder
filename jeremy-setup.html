Field Trips — Path-Finder Setup

This guide gets you from scratch to running the path-finder on your laptop and contributing to the shared database. Follow the steps in order — each one builds on the last.

⏱ Takes about 20–30 minutes total, mostly waiting for downloads. You only have to do this once.
1

Open Terminal — you already did this ✓

Terminal is a text window where you type commands. You found it already. Leave it open for the rest of this guide.

💡 How to use this guide: Each command block has a Copy button in the top-right corner. Click it, then click inside your Terminal window, and press ⌘V (Command+V) to paste. Then press Enter to run it.
⚠️ You may see a message about zsh
If Terminal says "The default interactive shell is now zsh", that's fine — ignore it and keep going.
2

Install Homebrew

Homebrew is a package manager — it's the easiest way to install developer tools on a Mac. Think of it as the App Store, but for programming tools.

Paste this into Terminal and press Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

This will ask for your Mac login password (the one you use to unlock your computer). Type it and press Enter — the characters won't show up as you type, that's normal.

It also may ask you to press Enter to confirm. It'll then download and install for a few minutes.

✅ How to know it worked:
When it's done, you'll see a line that says Installation successful! or you'll just get a new prompt like jeremyhinen$.

If your Mac is an Apple Silicon Mac (M1, M2, M3, or M4 chip — bought after late 2020), Homebrew installs in a different location and you'll need to run one more line after installation. If the installer tells you to run something that starts with eval, copy and paste that line and run it too.

✅ Verify Homebrew works:
Paste this and press Enter:
brew --version
You should see something like Homebrew 4.x.x. If you do, Homebrew is working.
❌ If brew --version says "command not found":
Your Mac is probably Apple Silicon. Run this:
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && eval "$(/opt/homebrew/bin/brew shellenv)"
Then try brew --version again.
3

Install Python and Git

Python is the programming language the path-finder is written in. Git is the tool that downloads and tracks code. Install both in one command:

brew install python@3.12 git

This takes 2–5 minutes. You'll see a lot of text scrolling by — that's normal.

✅ Verify:
python3.12 --version && git --version
You should see two lines, something like:
Python 3.12.x
git version 2.x.x
4

Create a GitHub account & get access Ask Sarah

The path-finder code lives on GitHub, a website for storing code. You need an account to download it.

  1. Go to github.com and click Sign up
  2. Use any email address and pick a username. Write it down — you'll need it.
  3. Complete the verification and confirm your email.
  4. Send Sarah your GitHub username — she'll add you to the Field Trips organization so you can access the private repo.
  5. Once Sarah adds you, you'll get an email from GitHub — click Accept invitation.
⚠️ Wait until you've accepted the invitation before going to Step 5.
5

Download the path-finder code

First, tell Git who you are. Replace the name and email with your own:

git config --global user.name "Jeremy Hinen"
git config --global user.email "your@email.com"

Now download (clone) the path-finder repo into a folder called field-trips on your computer:

mkdir -p ~/field-trips && cd ~/field-trips && git clone https://github.com/Field-Trips/path-finder.git && cd path-finder

When it asks for your GitHub username and password: the password is not your GitHub account password — it's a Personal Access Token. Create one like this:

  1. Go to github.com/settings/tokens/new
  2. Give it a name like field-trips
  3. Set expiration to No expiration
  4. Under "Select scopes", check repo
  5. Click Generate token
  6. Copy the token (it starts with ghp_) — save it somewhere safe, you can't see it again

Use that token as the password when Git asks.

✅ Verify:
ls ~/field-trips/path-finder/src/
You should see path_finder.py in the output.
6

Set up the Python environment

This creates an isolated space for the path-finder's dependencies (extra Python libraries it needs) and installs them:

cd ~/field-trips/path-finder && python3.12 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt

This takes 1–3 minutes and shows a lot of text — that's normal. You'll see "Successfully installed …" at the end.

✅ Verify:
cd ~/field-trips/path-finder && .venv/bin/python -m src.path_finder --help
You should see: Usage: python -m src.path_finder [OPTIONS] COMMAND [ARGS]...
7

Add your API keys Ask Sarah

The path-finder needs two secret keys to work — one to call Claude (AI that picks each Wikipedia hop), one to save results to the shared database. Sarah will send these to you.

Create the file that holds them:

cd ~/field-trips/path-finder && cp .env.example .env && open -e .env

This opens the file in TextEdit. You'll see:

ANTHROPIC_API_KEY= SUPABASE_URL=https://vjikcsifkvphuiwjrmqi.supabase.co SUPABASE_SERVICE_ROLE_KEY=

Fill in the two blank values with what Sarah sends you. The line should look like:

ANTHROPIC_API_KEY=sk-ant-api03-...your key here... SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...your key here...

Save the file (⌘S) and close TextEdit.

⚠️ Keep these keys private. Don't share them in email or Slack — send them via iMessage instead. The .env file is never uploaded to GitHub.
✅ Verify the database connection:
cd ~/field-trips/path-finder && .venv/bin/python -c " from dotenv import load_dotenv; load_dotenv(override=True) import os from supabase import create_client from supabase.client import ClientOptions sb = create_client(os.environ['SUPABASE_URL'], os.environ['SUPABASE_SERVICE_ROLE_KEY'], options=ClientOptions(schema='path_finder')) result = sb.table('nodes').select('id,title').limit(5).execute() print('Connected! Nodes in database:', len(result.data)) for n in result.data: print(' -', n['title']) "
You should see: Connected! Nodes in database: 3 (or more as the database grows).
❌ If you see an auth error:
The key in your .env file might have an extra space. Open the file again (open -e ~/field-trips/path-finder/.env) and make sure there's no space between the = and the key value.
8

Run your first path 🎉

Everything is set up. Run the interactive path-finder:

cd ~/field-trips/path-finder && python run.py

It will ask you three questions:

  1. Start Wikipedia article — type something like Monhegan
  2. End Wikipedia article — type something like Conservation movement
  3. Permutations — type 1 for your first test

Then it'll run for 30–90 seconds and show you the path it found, like:

[1] ✓ via coastal island ecology
    Monhegan, Maine → Protected area → Conservation movement

That path is now saved in the shared Field Trips database. You and Sarah are both contributing to the same dataset.

✅ You're fully set up.
From now on, whenever you want to run the path-finder, just open Terminal and type:
cd ~/field-trips/path-finder && python run.py

Common issues

Wikipedia page not found
The article title has to match Wikipedia exactly. Go to Wikipedia, find the article, and copy the title from the browser tab.

Path hits the hop cap (shows ×)
Some topics are far apart in Wikipedia's link graph. Try a different end article, or run with a higher hop limit by adding --max-hops 15 at the end of the command.

Terminal says "no such file or directory"
Make sure you've navigated to the right folder first: cd ~/field-trips/path-finder

The venv isn't activated
If you see an error about missing packages, run: source ~/field-trips/path-finder/.venv/bin/activate

Anything else
Send the full error message to Sarah.
